## 核实结论

1. **内部转岗是首选路径**：候选人现职百度云 ACG → IaaS → 镜像 / 软件源研发（2025.7 入职），即将满 1 年转岗窗口。百度内部 HR 系统（百度如流 / 内网）搜岗直接发起转岗，**完全免学历卡**（西安理工本科在外部是硬伤，内部完全不卡）= 最大护城河。外部社招投百度无意义（现员工走内部即可），且外部触发隐性 211 偏好。
2. **外部社招无对口岗**：talent.baidu.com 外部门户 SPA 代理渲染失败无法抓；Lagou 抓百度北京 113 社招岗（C++/Golang 本科，算法硕士，西二旗 / 西北旺 / 海淀），未找到"智能体平台 / Agent 架构"外部社招岗——核心岗走内部 / 猎头，必须内部转岗触达。
3. **目标方向匹配**：文心一言偏研究型算法，但**推理 Infra / 部署基础设施需底层系统工程师**——eBPF + 内核 + C 高度匹配推理 Infra 的可观测 / 性能 / 调度。目标团队排序：①大模型推理基础设施（推理引擎优化 / 服务化 / 可观测）> ②Agent 平台后端 > ③模型部署运维。
4. **通勤劣势**：百度成规模研发团队集中上地-西二旗-西北旺三角，距北医三院 8.4-10.4km（西二旗 ~9.7km）。转岗改不了通勤——转岗解决不了"离家近"，但换来了免学历卡 + 零跳槽成本。简历 / 面试强调"愿搬迁或弹性办公"（百度部分团队混合办公）。
5. **现职判断**：现职镜像 / 软件源底层支撑运维型，7 年底层高度匹配，大概率已是百度体系内"能躺且用得上底层能力"的较优解之一。转岗动机应聚焦"技术成长 / 推理 Infra 主航道"而非"现职不满"，避免现主管卡人。
6. **薪资**：T3-T8，7 年社招 C / 底层定级大概率 T5-T6。内部转岗一般平调平薪。转岗推理 Infra 主航道后成长性 / 涨薪空间 > 现职支撑线。

## 百度内部转岗行动指南（操作步骤，非邮件）

### ① 百度如流 / 内网 HR 系统搜岗关键词清单

> 百度 In（内部招聘系统）搜索，组合以下关键词逐个筛：

- **首选（推理 Infra 主航道）**：`大模型推理` / `推理引擎` / `推理服务化` / `推理基础设施` / `推理优化` / `Infra` / `高性能推理`
- **Agent 平台方向**：`智能体` / `Agent 平台` / `Agent 后端` / `智能体编排` / `文心智能体`
- **模型部署运维**：`模型部署` / `AI 平台` / `机器学习平台` / `推理运维` / `GPU 调度` / `大模型部署`
- **底层系统对口**：`性能优化` / `可观测` / `eBPF` / `系统工程师` / `C++ 后端` / `基础架构`
- **BU 过滤**：优先 ACG（云）/ Baidu Cloud / 智能云 / 文心（ERNIE）团队 / 大模型部；避开纯算法 / 研究院岗（硕士偏好 + 非系统侧）。

> **筛岗技巧**：看 JD 中是否出现"推理引擎 / serving / GPU / 性能 / 可观测 / 调度 / C++/Go 后端"——出现则系统侧对口；若全是"算法 / 论文 / NLP / 训练"则跳过。急招岗（标注"急" / HC 多）门槛低好进，优先。

### ② 目标团队排序

| 优先级 | 目标团队 | 匹配理由 | 备注 |
|---|---|---|---|
| **1** | 大模型推理基础设施（推理引擎优化 / 服务化 / 可观测） | eBPF + 内核 C + perf 直接对口推理性能 / 资源调度 / GPU 推理效率；百度云 IaaS 在职 | 主航道，成长 + 涨薪空间最大 |

### ④ 现主管沟通话术（如何提转岗不被卡）

> **时机**：2025.7 入职，建议 2026.7 满一年窗口后再正式提（部分团队要求满 1 年方可转岗）。提前 1-2 个月（即 2026.5-6）先口头试探，避免突袭。
>
> **话术（聚焦成长不抱怨）**：
> - "老大，入职快一年了，想跟你聊聊后续发展。我在镜像 / 软件源这边底层能力用得上，但个人更希望往大模型推理 Infra 主航道深耕——这块百度是核心战略，我的 eBPF / 内核 / 性能调优经验能直接用上，想申请内部转岗。"
> - **强调不跳槽**："不是要离开百度，是想在百度内部换到更对口的成长方向，现职积累的系统底层能力正好迁移过去。"
> - **承诺交接**："转岗前会把软件源 / 镜像这边的工作完整交接，保证不影响团队。"
>
> **避免**：不说"现职太躺 / 没挑战 / 涨薪慢"（触发主管危机感卡人）；不说"离家远想换办公点"（转岗改不了通勤，且暴露非工作动机）。聚焦"主航道 / 技术成长 / 百度内部长期发展"。
>
> **若被卡**：百度内部转岗一般主管不能强行阻拦（非竞业），但可拖延。策略：先拿到目标团队 Offer / 确认函再正式提，缩短主管反应窗口；必要时走 HRBP 协调。

## 操作步骤

1. **导出简历 PDF**：基于 `resume-大模型Infra版.md`，按上述突出点微调（强化"推理 Infra 可观测 / 性能"框定），导出 PDF 备用。
2. **百度如流搜岗（立即）**：登录百度 In / 如流，用①关键词清单逐个搜，筛出 3-5 个推理 Infra 对口岗，记录团队 / Leader / HireMgr / JD。
3. **联系目标团队（本周）**：如流直接发③话术给目标团队 Leader / HireMgr，附简历 PDF；同步在内部 HR 系统发起转岗申请。
4. **目标团队排序确认**：按②排序，优先大模型推理基础设施团队；若该方向无 HC，退回 Agent 平台后端 / 模型部署运维 / 同 BU 底层团队。
5. **现主管沟通（2026.5-6 满一年窗口前口头试探，2026.7 正式提）**：用④话术，聚焦主航道成长不抱怨现职，承诺交接。先拿目标团队确认函再正式提，缩短反应窗口。
6. **通勤对冲**：面试 / 沟通时主动表态愿弹性办公 / 搬迁，用⑤话术。
7. **跟进**：目标团队回复后安排内部面（通常简短 / 无面试），确认转岗 Offer → 现主管审批 → 双方确认 → 交接入职。
