## 一、静态分析 / 代码分析引擎（核心匹配）

- **BPF verifier 深度改造**：在 4.19 企业内核上回合高版本 bpf/btf 特性，核心工作是对 BPF **校验器（verifier）** 的分析与增强——verifier 本质是对 BPF 字节码做**静态安全分析**（路径分析、类型推导、范围检查），修复校验出错致近百用例失败。**这就是"深度理解代码语义与结构"在内核场景的实战**。
- **crash/perf 工具链长期实战**：用 crash 做内核 core dump 静态分析（反汇编→数据结构→调用链→根因），perf/ftrace 做动态 trace + 静态符号分析。迁移到代码分析引擎：程序分析（AST/CFG/调用图）、复杂度量化、根因定位的方法论完全一致。
- 定位过的疑难案例：申威 io_uring fork io 线程段错误（架构代码静态分析）、硬件随机单 bit 翻转 → do_page_fault（内存现场静态分析）、ARM 二层转发软中断占满 CPU（ebt_do_table 读写锁 + LSE 原子锁，锁竞争静态分析）。
