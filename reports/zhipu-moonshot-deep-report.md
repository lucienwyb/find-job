## 一、智谱 Zhipu（海淀中关村东路，距≈3-4km，已上市02513）

- 经验学历：本科及以上，3-7年，有推理框架或大模型Infra经验优先。**预估对硬核系统背景者放宽学历**。

### 1.3 智谱最稳投递切入岗位 + 面试核心讲点

**最稳切入：岗位B 大模型基础架构/系统软件工程师（Infra方向）**
- 理由：内核网络栈/内存/IO/eBPF/crash全栈直接对口集群OS层，匹配82、能进"高"；无需先补GPU即可投，入职后再补。

**面试核心讲点（把内核包装成大模型Infra加分项）**：
1. **"集群稳定性=内核稳定性"**：统信Kernel Panic/内存泄漏/性能瓶颈定位+eBPF/crash/kdump全栈→直接迁移到GPU节点内核panic定位、训练任务OOM/crash根因分析。讲"随机单bit翻转→do_page_fault宕机"案例=显存/内存ECC故障定位能力。
2. **"低延时网络调优→千卡通信调优"**：上交所ARM平台UDP 15w并发<100us、软中断绑核、中断聚合、gazelle/dpdk→迁移到NCCL/RDMA over RoCE通信调优、GPU集合通信性能优化。**这是最强迁移故事**，把网络栈低延时经验包装成"我知道如何让host侧数据通路不成为瓶颈"。
3. **"eBPF回合1000补丁→集群可观测性"**：bpf/btf机制深度+bcc/libbpf→集群eBPF监控、网络/IO profile、性能归因。
4. **"IO调优→并行存储"**：百度软件源重构(pulp分析、100T rsync同步IO瓶颈)→大模型训练checkpoint读写、并行文件系统(Lustre/weka)IO调优。
5. 主动提"补强计划"：已在学vLLM PagedAttention/CUDA基础，表明向推理系统方向迁移意愿——降低"无GPU经验"顾虑。

## 四、"从内核切进大模型"最短路径建议

**目标方向：推理系统优化（系统侧切入），从内核内存/IO/网络调优迁移到vLLM/TensorRT-LLM系统侧。**

### 4.1 经验迁移映射（内核 → 大模型Infra）

| 候选人已有 | 迁移到 | 大模型Infra对应 |
|-----------|--------|----------------|
| 内核内存(伙伴/SLAB/页对齐/memcg) | → | KV cache显存管理、PagedAttention、host显存换入换出 |
| 网络栈低延时调优(UDP<100us/软中断绑核/dpdk) | → | NCCL/RDMA over RoCE通信调优、集合通信性能 |
| eBPF(bcc/libbpf)+btf回合1000补丁 | → | 集群可观测性、网络/IO/GPU profile、性能归因 |
| crash/kdump/ftrace排障 | → | GPU节点内核panic定位、训练OOM/crash根因 |
| IO调优(百度软件源100T同步/pulp) | → | checkpoint读写、并行文件系统(Lustre/weka)IO调优 |
| kpatch热补丁产品化 | → | 集群节点热修复、零停机运维 |

### 4.2 最短补强清单（按优先级，1-2月可上手面试）

1. **vLLM源码（最高优先）**：精读PagedAttention、KV cache分块、continuous batching、prefix cache。本地跑vLLM，profile瓶颈，写1个小优化PR/demo（如KV cache eviction策略、显存复用）。这是面试推理系统岗的"投名状"。
2. **CUDA/Triton基础**：2-3周入门。理解SM/warp/thread层级、显存层级(global/shared/register)、kernel launch开销。会写简单kernel、能读CUDA C++代码。不需要精通，能对话即可。
3. **NCCL/RDMA原理**：从内核网络栈角度理解RoCE/IB、GPUDirect RDMA、NCCL all-reduce拓扑。看NCCL源码通信原语。
4. **K8s基础（系统岗/SRE岗）**：1-2周。Pod/Service/Device Plugin( NVIDIA GPU Operator)/调度器概念。实操Minikube跑通。
5. **PyTorch基础**：1周。会读训练loop、理解dataloader/distributed。
6. **GPU诊断工具**：nvidia-smi/DCGM/nsys/ncu，2-3天熟悉。

### 4.3 切入策略

- **不要先补全GPU再投**：系统侧岗位（B/C）现在就能投，内核经验是敲门砖；入职后在GPU集群实战中补GPU。**先入职、再纵深**。
- **首选智谱系统软件岗(B)**：通勤近、强度可控、学历门槛低、匹配82、能进"高"。
- **保底月之暗面SRE(C)或系统(B)**：走内推，简历前置硬核战绩。
- **避开**：算法岗、纯CUDA/Triton kernel岗、GPU集群(D)(GPU硬要求高)、训练框架岗（分布式训练经验缺）。

### 4.4 简历改造建议（针对两家）
