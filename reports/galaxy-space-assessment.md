## 3. 最稳投递切入岗位 + 面试核心讲点

### 切入岗位：**应用软件工程师（嵌入式）**（岗位A）为主，**驱动软件开发工程师**（岗位B）为辅

**主投A理由**：闻翔Zynq经历 = 星载载荷计算单元标准画像，JD要求的"嵌入式平台(ARM/PowerPC)编程+嵌入式OS开发+星载应用软件设计/测试/维护"几乎逐项命中。岗位A是"应用+底层通吃"岗，候选人闻翔经历+统信内核覆盖完整。

**辅投B理由**：驱动岗门槛低(JD只要1年Linux驱动)，候选人4年内核远超；上交所UDP低延时+网卡中断内核模块(300行)直接对口"星载5G网络产品性能优化"。若A无headcount或团队更偏通信，B是稳的备选。

### 面试核心讲点（按简历项目串联）

1. **闻翔Zynq全栈移植**（对照岗位A的"基于星载硬件平台做星载应用软件"）
   - Petalinux移植uboot/kernel/rootfs全流程 → 证明能独立把新SoC的嵌入式Linux跑起来（星载平台入职第一件事）
   - SPI/LCD/触摸屏/PL UartLite设备树与驱动 → 设备树+字符驱动+platform driver扎实
   - AD9361射频驱动移植 → 银河航天Q/V/Ka载荷射频链路直接相关（AD9361是软件无线电经典芯片，星载通信载荷常见）
   - FPGA控制代码重构7000行 → PS-PL AXI交互、寄存器/DMA协同、能读FPGA代码并能改
   - 雷达目标模拟器0.15m精度 → 实时性/串口分包/中断接收工程能力

2. **统信4年内核疑难Bug方法论**（对应星载OS稳定性/在轨维护）
   - ftrace/perf/eBPF/crash四件套定位链路，方法论清晰
   - **随机单bit翻转→do_page_fault宕机排查**：⭐绝杀——直接引出"空间单粒子翻转(SEU)对星载软件的影响、EDAC/内存冗余/TMR的必要性"，把地面硬件经验翻译成空间可靠性意识。候选人独有的差异化故事。
   - 申威io_uring fork io线程段错误（架构代码）→ 陌生架构/陌生子系统深挖根因
   - ARM二层转发软中断占满CPU（ebt_do_table读写锁+LSE原子锁）→ 网络栈锁机制深度
   - ARM RAS调研+内存错误注入实验 → 直接对口星载RAS/EDAC
   - kpatch热补丁产品化（多版本rpm spec打包）→ 直接对口星载在轨软件上注/热补丁

3. **上交所UDP低延时调优**（对应岗位B"星载5G网络产品性能优化"）
   - 15万并发<100us、软中断绑核、中断聚合、gazelle/dpdk适配 → 用户态网络/极致低时延，星间链路通信软件直接受益
   - bpf/btf回合~1000补丁 → 网络栈/eBPF深改能力，不止"用"还会"改内核"
   - 独立写网卡中断信息展示内核模块(~300行) → 直接对应"Linux平台内核及驱动开发"

4. **应用物理学背景**（隐性加分）
   - 射频/电磁/信号处理有物理直觉，理解AD9361/相控阵/波束成形背后的物理量，比纯CS背景更易沟通载荷射频团队

5. **短板主动坦白 + 学习意愿**
   - 航天标准(ECSS/QJ)、抗辐照(SEU/SEL/TID)、测控链路未系统接触 → 但bit翻转排查经历说明已有问题意识，可快速补
   - VxWorks/RTEMS未做过项目 → Linux精通可迁移，RTOS API差异小
   - PREEMPT_RT/Xenomai硬实时未做过项目 → 通用内核调优经验可迁移

## 4. 银河航天加班文化 / 工作强度判断

**整体判断：中等，官方双休+弹性，发射节点有突击**

**真实数据来源**：
- 猎聘官方公司画像标注：**周末双休、弹性工作、股票期权、绩效奖金、带薪年假、五险一金**
- 官方宣称：现金薪酬"超互联网行业水平，航天科技类岗位对标BATH"；有股票期权；"更高机会为应届生提供北京户口"

**行业判断**：
- 商业航天行业共性：卫星研发长周期工程，非互联网"7*24上线"节奏。常态大致9:30-20:30双休为主，少数团队大小周。
- 发射节点突击：卫星发射窗口前后1-2个月，AIT/试验/在轨测试阶段会有集中加班（含周末），航天行业必然现象，候选人需心理预期。
- 银河航天作为头部民营商业航天（已发多颗卫星+自研百颗级生产线+启动IPO），比纯初创规范，比体制内（航天科技/科工集团）节奏快。
- 星载软件岗相比火箭测发岗、AIT现场岗，软件岗强度相对可控，加班更集中于版本节点/联调节点。

**与候选人"不忙"偏好的契合**：在4家重点公司里（银河航天/银河通用/智谱/月之暗面），银河航天是强度最接近"不996/不大周"要求的——大模型公司以高强度著称，机器人初创迭代紧。航天软件岗是相对"有节奏感"的工程岗。猎聘标注"双休+弹性"是正向信号。

**建议**：面试时主动确认目标团队是否大小周、发射周期加班强度。星载软件团队通常比AIT/测发团队规律。
