## 1. 可匹配岗位清单（5个，按匹配度排序）

### 岗位A：星载软件工程师 / 星载嵌入式软件工程师（载荷计算方向）⭐最对口
**典型职责**（行业推断）
- 低轨宽带卫星载荷计算单元（Zynq SoC / ARM+FPGA / 国产化SoC如龙芯/飞腾）板级BSP与嵌入式Linux裁剪定制
- 载荷软件在轨运行框架、星载业务应用（路由/通信处理）开发
- 星载软件全生命周期：方案、设计、编码、仿真测试、AIT联调、在轨维护
- 与载荷FPGA联调（PS-PL AXI接口、寄存器、DMA）
- 在轨故障定位与软件重构/补丁上注

**技术栈**
- 语言：C为主，C++次之，少量汇编
- 平台：ARM Cortex-A、Zynq/SoC、Petalinux/Yocto嵌入式Linux；VxWorks/RTOS了解加分
- 工具：交叉编译、GDB、JTAG、示波器/频谱仪、Git
- 加分：Zynq PL侧交互、AD9361等射频芯片驱动、AXI/DMA

**经验/学历**：本科3-7年，航天背景优先但非强制（看团队）
**薪资范围（北京商业航天行情，预估）**：25-45k×14-16薪，年总包35-70万。资深/骨干可到60-80万。
**是否需航天背景**：否，看项目落地能力。

---

### 岗位B：星载操作系统 / 系统软件工程师（星载Linux方向）⭐对口
**典型职责**
- 星载嵌入式Linux/实时OS定制、裁剪、移植到星载SoC
- 内核稳定性/实时性/容错：PREEMPT_RT、看门狗、内存隔离、抗单粒子翻转
- 内核驱动开发与维护（网口、Flash、看门狗、EDAC、RAS）
- 在轨软件热补丁、在线升级、故障注入与可靠性验证
- 内核疑难问题定位（Panic、死锁、内存泄漏、性能瓶颈）

**技术栈**
- 语言：C精通
- 平台：ARM/x86/国产SoC内核，Linux mainline + 4.19/5.10/6.x LTS
- 工具：ftrace、perf、eBPF/bcc/libbpf、kdump/crash、SystemTap
- 加分：PREEMPT_RT/Xenomai、kpatch/livepatch、ARM RAS/EDAC、看门狗

**经验/学历**：本科5年以上，资深15-40k×14-16薪→年总包25-60万（预估）
**是否需航天背景**：否，内核功底为主。空间可靠性意识可入职后补。

## 3. 最稳投递切入岗位 + 面试核心讲点

### 切入岗位：**星载软件工程师 / 星载嵌入式软件工程师**（岗位A）

理由：闻翔Zynq经历 = 星载载荷计算单元标准画像，几乎逐项命中JD；统信内核经验覆盖星载OS稳定性/热补丁需求；"随机bit翻转宕机排查"是切入空间单粒子翻转话题的绝佳故事。

### 面试核心讲点（按简历项目串联）

1. **闻翔Zynq全栈移植**（最强讲点，逐项对照星载载荷平台）
   - Petalinux移植uboot/kernel/rootfs全流程 → 证明能独立把一个新SoC的嵌入式Linux跑起来（星载平台入职第一件事）
   - SPI/LCD/触摸屏/PL UartLite设备树与驱动 → 证明设备树+字符驱动+platform driver扎实
   - AD9361射频驱动移植 → 银河航天载荷射频链路直接相关（AD9361是软件无线电经典芯片，星载通信载荷常见）
   - FPGA控制代码重构7000行 → 证明PS-PL AXI交互、寄存器/DMA协同、能读FPGA代码并能改
   - 雷达目标模拟器0.15m精度 → 证明实时性/串口分包/中断接收的工程能力

2. **统信4年内核疑难Bug方法论**（对应星载OS稳定性/在轨维护）
   - ftrace/perf/eBPF/crash四件套定位链路，方法论清晰
   - **随机单bit翻转→do_page_fault宕机排查**：⭐绝杀——直接引出"空间单粒子翻转(SEU)对星载软件的影响、EDAC/内存冗余/TMR的必要性"，把地面硬件经验翻译成空间可靠性意识。这是候选人独有的差异化故事。
   - 申威io_uring fork io线程段错误（架构代码）→ 证明能在陌生架构/陌生子系统深挖到根因
   - ARM二层转发软中断占满CPU（ebt_do_table读写锁+LSE原子锁）→ 网络栈锁机制深度
   - ARM RAS调研+内存错误注入实验 → 直接对口星载RAS/EDAC
   - kpatch热补丁产品化（多版本rpm spec打包）→ 直接对口星载在轨软件上注/热补丁

3. **上交所UDP低延时调优**（对应星间/星地链路低时延需求）
   - 15万并发<100us、软中断绑核、中断聚合、gazelle/dpdk适配 → 证明能做用户态网络/极致低时延，星间链路通信软件直接受益
   - bpf/btf回合~1000补丁 → 证明对网络栈/eBPF有深改能力，不止"用"还会"改内核"

4. **应用物理学背景**（隐性加分）
   - 射频/电磁/信号处理有物理直觉，理解AD9361/相控阵/波束成形背后的物理量，比纯CS背景更易沟通载荷射频团队

5. **短板主动坦白 + 学习意愿**
   - 航天标准(ECSS/QJ)、抗辐照(SEU/SEL/TID)、测控链路未系统接触 → 但bit翻转排查经历说明已有问题意识，可快速补
   - PREEMPT_RT/Xenomai硬实时未做过项目 → 通用内核调优经验可迁移，愿意自学做demo

## 4. 银河航天加班文化 / 工作强度判断（行业推断）

**整体判断：中等偏忙，发射节点有突击，非普遍996**

依据：
- **商业航天行业共性**：处于快速迭代期，星座组网节奏紧，但卫星研发是长周期工程，非互联网"7*24上线"节奏。常态大致是 9:30-20:30，双休为主，少数团队大小周。
- **发射节点突击**：卫星发射窗口前后1-2个月，AIT/试验/在轨测试阶段会有集中加班（含周末），这是航天行业必然现象，候选人需心理预期。
- **银河航天具体情况**：作为头部民营商业航天（估值数百亿、已发多颗卫星含中国首颗低轨宽带通信卫星、自研卫星批量生产线），比纯初创规范，比体制内（航天科技/科工集团）节奏快。研发岗普遍单休大小周的传闻较少，更接近"常态加班+节点突击"。
- **星载软件岗**：相比火箭测发岗、AIT现场岗，软件岗强度相对可控，加班更可控于版本节点/联调节点。
- **与候选人"不忙"偏好的契合**：在4家重点公司里，银河航天是强度最接近候选人"不996/不大周"要求的——大模型公司(月之暗面/智谱)以高强度著称，机器人初创(银河通用)迭代紧。航天软件岗是相对"有节奏感"的工程岗。

**建议**：面试时主动确认目标团队是否大小周、发射周期加班强度。星载软件团队通常比AIT/测发团队规律。

## 5. 投递行动建议

1. **第一优先投递**：银河航天官网/拉勾/Boss搜"星载软件""嵌入式软件""星载Linux""系统软件"岗，主投岗位A，备投岗位B。
2. **简历包装**：把闻翔Zynq经历提到最前（现简历已较前），明确标注"Petalinux移植/设备树/驱动/AD9361/FPGA重构"关键词；统信经历突出"随机bit翻转排查""kpatch产品化""ARM RAS"三条与星载直接相关。
3. **预期薪资**：参考行情开30-40k×15薪（年总包45-60万），视面试评级上下浮动。百度云现职若IaaS镜像岗薪资偏低，银河航天通常能给出竞争力涨幅。
4. **学历风险**：航天工程岗对985/211不如大模型公司敏感，本科+7年经验+对口项目可过初筛。
5. **风险提示**：若银河航天该批次星载软件headcount已满或侧重信号处理（DSP/FPGA为主），可能被引导到岗位D/E，需评估是否接受。
